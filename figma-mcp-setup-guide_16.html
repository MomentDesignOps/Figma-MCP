02

Verify Node.js Installation

Confirm Node.js is correctly installed — should show v18 or higher
Open Terminal and run the version check

Press Win + R on your keyboard → type cmd in the Run dialog → press Enter.

Open Spotlight with ⌘ + Space → type Terminal → press Enter.

Then enter the following command:

Terminal / Command Prompt 
node --version
✅ If it shows v18.x.x or higher (e.g. v20.11.0), you're good to proceed. If you see an error or a lower version, raise a SN Request for Node.js installation.
03

Get Your Figma Access Token

Required so the MCP server can read your Figma files
Create a Personal Access Token in Figma

In Figma Desktop or browser, go to your account settings and generate a token by following these steps:

  1. Open Figma → click your profile picture (top-left)
  2. Go to Settings → click the "Security" tab
  3. Scroll down to the Personal Access Tokens section → click "Generate new token"
  4. Give it a name, e.g. claude-mcp
  5. Set the Expiration to the maximum available — 90 Days
  6. Under Scopes, enable all access by checking every checkbox
  7. Click Generate token and copy it immediately
⚠️ You won't be able to see this token again after closing the dialog. Store it somewhere safe (password manager, Notepad). The token always starts with figd_.
🚨 Do not share your Figma token with anyone — it grants access to your personal Figma files. Keep it strictly confidential.
04

Configure Your MCP Client

Add your Figma token to Claude Desktop's config — takes under 2 minutes
  1. Open the Claude App
  2. Click the Top Left Collapse Sidebar Icon → Click on your Profile Name → go to Settings
  3. Navigate to the last tab at the bottom named "Developer"
  4. Click on the Edit Config button
  5. Open the claude_desktop_config.json file using any available text editor
  6. Copy the below entire code and replace it in the file
  7. Copy your Figma token and paste it in place of figd_YOUR_TOKEN_HERE — make sure the token remains within double quotes " "
claude_desktop_config.json 
{
  "mcpServers": {
    "figma-console": {
      "command": "npx",
      "args": ["-y", "figma-console-mcp@latest"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "figd_YOUR_TOKEN_HERE",
        "ENABLE_MCP_APPS": "true"
      }
    }
  }
}
💡 Save the file after editing (Ctrl+S on Windows, ⌘+S on Mac). The JSON must be valid — no trailing commas or missing quotes.
05

Verify Figma Connection Status

Restart Claude and confirm the MCP server is running
1
Completely close Claude Desktop

Right-click the Claude icon in the System Tray (bottom-right of your taskbar) → click Quit. Do not just close the window.

Press ⌘ + Q to fully quit Claude. Do not just close the window — the app must be fully terminated.

2
Reopen Claude and check the Developer settings

After restarting, Claude will attempt to connect to Figma using the token provided. Follow these steps to check the status:

  1. Open the Claude App
  2. Click the Top Left Collapse Sidebar Icon → Click on your Profile Name → go to Settings
  3. Navigate to the last tab at the bottom named "Developer"
  4. Look for figma-console in the list and check its status
  5. Once figma-console shows "Running", check for any other Figma connectors in the list — if found, Disable them from the Connectors tab in Settings to avoid conflicts while designing
✅ If the status shows "Running" — congratulations! Your Figma is connected to Claude. You can now proceed to the next step.
⚠️ If it shows an error, double-check that your token in the config file is correct and that the JSON is valid. Make sure the token starts with figd_.
06

Setup Figma Desktop Bridge Plugin

Enables real-time write access and live design creation inside Figma
1
Download and extract the plugin files
  1. Download the plugin file and save it to your system
⬇ Download figma-console-mcp-main.zip
  1. Extract the folder to an easy-to-find location, e.g. your Desktop or Documents folder
2
Import the plugin into Figma Desktop
ℹ️ You must use Figma Desktop App for this step. The browser version of Figma will not work.
  1. Open Figma Desktop and open any New or Existing Figma file
  2. Go to Dev Mode and click on the "Enable Desktop MCP Server" Button
  3. Click the Figma logo icon (top-left corner)
  4. Go to Plugins → Development → click "Import plugin from manifest…"
  5. Navigate to the extracted figma-console-mcp-main folder
  6. Go into the figma-desktop-bridge subfolder
  7. Select the manifest.json file and click Open
  8. Once you receive the "Figma Desktop Bridge has been Imported" message, close the Figma Panel and open the plugin from Main Menu → Plugins → Development → Figma Desktop Bridge
✅ Once imported, you'll see "Figma Desktop Bridge" appear in your Plugins section.
3
Run the Desktop Bridge plugin
  1. Go to Plugins → find and click "Figma Desktop Bridge"
  2. The plugin panel will open — wait for the status to show "MCP Ready"
✅ Once you see "MCP Ready", proceed to the next step. Do not close the plugin — keep it running in the background while using Claude.
07

Check Figma Status & Test

Confirm the connection and create your first AI-generated frame
1
Check the Figma connection from Claude

With the Desktop Bridge plugin showing "MCP Ready" in Figma, switch to Claude Desktop and open a new chat. Enter this prompt:

Claude Chat Prompt 
Check Figma status
✅ Once Claude responds with "Figma is Connected", you're ready to start designing with AI.
2
Create your first frame with Claude

In the same Claude chat, test the Figma write access with this prompt:

Claude Chat Prompt 
Create a simple frame with a blue background
🎉 If a frame appears directly in your Figma file — you're all set! Claude is now fully connected to your Figma and can read and create designs.
